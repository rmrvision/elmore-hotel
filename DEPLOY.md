# Деплой El More на Timeweb Cloud Apps + S3

Архитектура:
- **Сайт + админка** → Timeweb App Platform (Node.js)
- **Контент (`content.json`) и загруженные фото** → Timeweb S3-хранилище
- Правки из админки переживают редеплои, т.к. лежат не в контейнере

---

## Шаг 1 — Создать S3-бакет

1. Зайдите в [панель Timeweb Cloud](https://timeweb.cloud/my/storage) → **«Объектное хранилище» (S3)** → **Создать**.
2. Параметры:
   - **Имя бакета:** например, `elmore-storage` (имя должно быть уникальным в рамках Timeweb).
   - **Тип:** **Публичный** (`public`). Это позволит браузерам напрямую загружать фото с S3 по URL.
   - **Тариф:** минимальный, для отельного сайта хватит надолго.
3. После создания зайдите в бакет → раздел **«Доступ»** или **«S3-ключи»**.
4. Создайте пару ключей доступа → **скопируйте `Access Key ID` и `Secret Access Key`**. Secret показывается только один раз, сохраните.

> Регион в Timeweb сейчас один — `ru-1`, endpoint — `https://s3.twcstorage.ru`. Их менять не надо.

## Шаг 3 — Создать приложение в App Platform

1. [Панель Timeweb](https://timeweb.cloud/my/apps) → **App Platform → Создать**.
2. **Тип:** Backend → **Фреймворк:** Express (или Node.js).
3. **Репозиторий:** GitHub → `rmrvision/elmore-hotel`, ветка `main`.
4. **Команда сборки:** `npm install && npm run build`
5. **Команда запуска:** `npm start`
6. **Версия Node.js:** 22.
7. **Переменные окружения** — добавить все:

| Переменная | Значение | Примечание |
|---|---|---|
| `NODE_ENV` | `production` | |
| `PORT` | `3000` | Timeweb обычно сам пробрасывает |
| `ADMIN_PASSWORD_HASH` | `$2a$12$...` | См. ниже |
| `SESSION_SECRET` | случайная строка | См. ниже |
| `S3_ENDPOINT` | `https://s3.twcstorage.ru` | |
| `S3_REGION` | `ru-1` | |
| `S3_BUCKET` | `elmore-storage` | имя вашего бакета |
| `S3_ACCESS_KEY` | `...` | из шага 1 |
| `S3_SECRET_KEY` | `...` | из шага 1 |

8. **Создать**. Через 2–4 минуты получите URL вида `https://elmore-xxxxx.twc1.net`.

### Как получить значения

**`ADMIN_PASSWORD_HASH`:**
```bash
# Локально установите пароль:
npm install
npm run setpass
# Скопируйте значение ADMIN_PASSWORD_HASH из .env
cat .env | grep ADMIN_PASSWORD_HASH
```

Или одной командой (замените «ваш-пароль»):
```bash
node -e "console.log(require('bcryptjs').hashSync('ваш-пароль', 12))"
```

**`SESSION_SECRET`:**
```bash
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```

## Шаг 4 — Что произойдёт автоматически

При первом запуске сервер:
1. Подключится к вашему S3-бакету.
2. Проверит наличие `content.json` в бакете.
3. Если нет — загрузит стартовый `content.json` из репозитория в бакет.
4. Дальше всегда читает и пишет в S3.

Загружаемые через админку фото идут в S3 в папку `uploads/` с публичным ACL — браузер видит их по прямому URL вида:
```
https://s3.twcstorage.ru/elmore-storage/uploads/1715000000-foto.jpg
```

Бэкапы `content.json` хранятся в S3 в `backups/` (последние 20 версий).

## Шаг 5 — Кастомные домены

В Timeweb App Platform → вкладка **«Домены»**:
1. Добавьте `elmorehotel.ru` (основной).
2. Добавьте `admin.elmorehotel.ru` (субдомен админки).
3. У регистратора домена пропишите CNAME-записи на технический домен Timeweb (платформа подскажет какой).
4. SSL автоматически выпустится через Let's Encrypt.

Серверная логика субдоменов уже встроена:
- На `admin.elmorehotel.ru` корень `/` редиректит в `/admin`.
- На `elmorehotel.ru` отдаётся обычный сайт.

## Локальная разработка

В локальном режиме S3 не нужен — переменные `S3_*` оставляете пустыми, сервер автоматически переходит на файловую систему:
- `content.json` — в корне проекта
- `uploads/` — в корне проекта
- `backups/` — в корне проекта

```bash
npm install
cp .env.example .env
npm run setpass            # задать пароль
npm start
```

Сайт: `http://localhost:3000` · Админка: `http://localhost:3000/admin`

Если хотите локально тестировать со «своим» S3 — заполните `S3_*` в `.env`, сервер переключится в S3-режим.

## Обновление и откат

- **Обновить сайт:** запушить в `main` — Timeweb пересоберёт автоматически.
- **Откатить контент:** список бэкапов в S3 в папке `backups/`. Скачать нужный через панель Timeweb или CLI → загрузить вручную как `content.json`.
- **Сменить пароль админа:** `npm run setpass` локально → скопировать новый `ADMIN_PASSWORD_HASH` → обновить переменную в App Platform → перезапустить.
